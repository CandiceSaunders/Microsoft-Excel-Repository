# Agile Delivery & Operational Metrics Dashboard

<h2>Project Overview</h2>
This project simulates Agile delivery and operational reporting workflows using a custom-built dataset modeled after Agile and PMO environments. The dashboard was developed in Excel to analyze sprint throughput, workload distribution, backlog health, cycle time trends, and SLA performance across multiple teams and work item types.

<h2>Business Problem</h2>
Organizations managing Agile delivery and operational workflows require visibility into team workload, sprint performance, backlog trends, and SLA compliance to support delivery planning and operational decision-making.

<h2>Objectives</h2>

- Track Agile delivery KPIs
- Analyze sprint throughput trends
- Monitor backlog and workload distribution
- Identify SLA breach patterns
- Create interactive stakeholder reporting

<h2>Tools Used</h2>

- Microsoft Excel
- Pivot Tables
- Pivot Charts
- Slicers
- KPI Reporting
- Data Validation
- Dashboard Design

<h2>Key Metrics</h2>

- Total Work Items
- Open vs Closed Work Items
- Sprint Velocity
- Cycle Time
- SLA Breaches
- Monthly Work Item Trends
- Team Workload Distribution

<h2>Dashboard Features</h2>

- Interactive slicers
- KPI cards
- Trend analysis
- Operational reporting visuals
- Agile delivery metrics
- Dynamic filtering

<h2>Key Insights and Reccomendations</h2>

| Insight | Impact | Recommendation |
| -------- | -------- | -------- |
| Low priority items had longest cycle time | Aging backlog risk | Reassess prioritization |
| Reporting team carried highest workload | Potential bottleneck | Evaluate resource allocation |
| Sprint throughput fluctuated | Delivery inconsistency | Improve sprint planning |
| SLA breaches represented about 19% of work | Operational performance concern | Monitor aging items proactively |
